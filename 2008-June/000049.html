High Leute,

habe mit dem Script kwquota angefangen. Es unterstützt auch das XFS
Dateisystem. Als tools werden quotatool, xfs_quota und repquota
genutzt. Ein/Ausschalten/Dump/restore von Quota ist noch nicht
implementiert. 

Wer das Script mal anschauen möchte es ist im cvs.

Was geht:
- Anzeige von 1 oder mehreren Quota Laufwerken.
- Erstellen/verändern/löschen von Quota für Benutzer, geht
  hervorragend.
- Erstellen/verändern/löschen von Quota für Gruppen, will auf
  Debian/Lenny nicht. Sollte aber gehen da es nichts anderes als für
  Benutzer ist. Die Option -u (User) wird nur gegen -g (Group)
  getauscht.
- Erstellen/verändern/löschen von Quota für Projekte (XFS), noch nicht
  implementiert. Dürfte bei xfs nichts anderes als für
  Gruppen/Benutzer sein, bloss das noch 2 Dateien extra angelegt
  werden.

kwquota erzeugt beim Start die Dateien /Quota_Dev/(a)quota.user und
/Quota_Dev/(a)quota.group falls diese noch nicht existieren und das
Quota gerade aktiviert wurde (fstab,remount,...).

Bisher wurde alles auf Debian/i386 Lenny Laptop getestet.

changelog bisher:
kwtools (0.7.2-0) unstable; urgency=low
  * kwgroup/kwuser: add new options and display.
  * kwquota: create.
  * function read_fstab: simplified.
  * function search_packages: new variables NO_PACKAGES(_E).
  * new function no_package_inst_msg.

Für die kwxen-tools sollte ich vielleicht auf xen-3.2 updaten. Zur
Zeit meckert xen beim start über irgendeine python Datei. Na ja,
wird schon. kwquota ist wichtiger, da hier ab und zu einige User die
Home Partition (ausversehen) zu knallen.

Doku, cvs, manpages sind aktuell.
